自行利用 Ubuntu 建立可啟動 CMSiMDE 動態系統與 Fossil SCM 伺服器的目的:

建立完整的 Python 網際程式開發環境.

開發的程式碼可採用 Fossil SCM 與 Github 同步的方式進行分散式版次管理.

Ubuntu 20.04 on Virtualbox, 從 http://a.kmol.info:88/Ub2004.vdi 取得 已經安裝 Ubuntu 的虛擬硬碟進行後續設定. 或者自行下載 Ubuntu 20.04 Desktop 版 iso 檔案自行安裝帶有桌面的虛擬主機.

先更改 vdi 的 uuid:

"c:\Program Files\Oracle\VirtualBox\VBoxManage.exe" internalcommands sethduuid Ubwcm2021.vdi

設定虛擬主機網路連線:

接著將 Ubuntu 虛擬主機的網路以 Bridged 串接到 Windows 10 已經連線的網路卡.

登入系統後進行更新:

由於 /etc/netplan/net.yaml 中採用 DHCP6 連外, 因此透過 ifconfig 可以取得網路連線資訊.

但因電腦輔助設計室採純 IPv6 設定, 因此必須透過雙網路協定代理主機上網才可執行下列 apt 指令.

因此要以 sudo vi  /etc/apt/apt.conf 建立 apt 代理主機設定檔, 內容如下:

Acquire::http::Proxy "http://proxy_username:proxy_password@[Proxy_server_ipv6_address]:3128";

之後就可以完成下列更新指令執行:

sudo apt update

sudo apt upgrade

sudo apt autoremove

進行 nginx 與 uwsgi 的設定:

目的是讓 CMSiMDE 的動態與靜態系統可以在 Ubuntu 操作系統中以服務的方式, 隨電腦開機啟動.

首先安裝 nginx:

sudo apt install nginx

Windows 10 與虛擬主機交換檔案:

由於 Ubuntu 虛擬主機並無安裝 ubuntu-desktop, 因此並無 X Window GUI 介面, 因此要在 Windows 10 與純 IPv6 連外的虛擬主機交換檔案的方式, 採取內建一個內部 IPv4 區域網路進行資料交換. 意即將 /etc/netplan/net.yaml 設為:

# This is the network config written by 'subiquity'
network:
  ethernets:
    enp0s3:
      dhcp4: false 
      addresses:
        - 192.168.19.1/24
      gateway4: 192.168.19.254    
      dhcp6: true
      nameservers:
              addresses:
                      - 2001:b000:168::1

  version: 2

然後以 sudo netplan apply 啟用, 接著在 Windows 設定同群 IPv4 網路位址, 以便利用 python3 -m http.server 從 Ubuntu 啟動 www server, 讓 Windows 10 的瀏覽器可以直接取得 Ubuntu 系統中的檔案, 然後再從 Windows 10 以 sftp 對 Ubuntu 連線, 將修改後的相關設定檔案送進 Ubuntu, 然後透過 sudo cp 將設定檔案送至指定位置, 並令其生效.

使用者也可以利用 ifconfig > ifconfig 將 Ubuntu 網路設定檔案內容存為 ifconfig, 並以上述 Ubuntu www server 對 Windows 10 瀏覽器的連線方式將 ifconfig 設定檔案取下.

在 Ubuntu 可以透過 python3 -m http.server --bind ipv4_or_ipv6_address, 啟動 port 為 8000 的 www server.

隨後有關 nginx 或 uwsgi 的相關設定檔案, 就可以利用上述方式將設定檔案, 傳至 Windows 以圖形介面編輯修改後, 再傳回 Ubuntu 進行測試.

安裝 pip3:

假如 Ubuntu 尚未安裝 pip3, 可以利用 sudo apt install python3-pip 安裝.

安裝 CMSiMDE 所需 Python 模組:

由於目前所安裝的 Ubuntu 並無 GUI 介面, 因此並不會直接在 Ubuntu 操作系統中啟動 Leo Editor, 錦會透過 uwsgi 啟動動態系統, 以 nginx 伺服靜態系統網頁, 因此只需安裝:

sudo pip3 install flask bs4 lxml flask_cors

接下來安裝 uwsgi 所需程式庫與模組:

sudo apt install uwsgi uwsgi-plugin-python3

sudo pip3 install uwsgi

接下來希望將 https://github.com/mdecourse/cmstemplate clone 至 /home/wcm2021/github 目錄下, 其他目錄資料可利用 rm -r 目錄名稱 刪除, -r 代表連全部的子目錄一併刪除.

若希望使用 git clone --recurse-submodules https://github.com/mdecourse/cmstemplate.git 取下 cmstemplate 倉儲內容, 必須使用 git config 設定 http.proxy.

git config http.proxy http://proxy_account:proxy_password@[proxy_ipv6_address]:port

此時必須確認是否先前已經在 /home/wcm2021/ 目錄下已經設定 .gitconfig, 上述 git config 並無法蓋過先前設定的檔案, 必須以 vi .gitconfig 進入修改, 加上 proxy 帳號與密碼.

以 nginx 啟動靜態系統:

已知 cmstemplate 位於 /home/wcm2021/github/cmstemplate, 接下來希望利用 nginx 啟動此一靜態系統.

只要將 /etc/nginx/sites-available/default 中 port 80 伺服 root 設定為 /home/wcm2021/github/cmstemplate 即可由 nginx 伺服 index.html.

#root /var/www/html;
root /home/wcm2021/github/cmstemplate/;

然後利用 sudo /etc/init.d/nginx restart 重新啟動 nginx.

以 nginx 啟動動態系統:

uwsgi 與 nginx 結合, 需要 /home/wcm2021/uwsgi_ini 目錄中的 uwsgi.ini 設定檔案與 /etc/nginx/sites-available/default 中的 port 443 server 設定:

server {
    listen 443 ssl default_server;
    listen [::]:443 ssl default_server;
   
    location / {
        include uwsgi_params;
        uwsgi_pass  127.0.0.1:8080;
    }
   
    ssl_certificate /etc/stunnel/localhost.crt;
    ssl_certificate_key /etc/stunnel/localhost.key;
    ssl_session_timeout 5m;
    ssl_protocols SSLv3 TLSv1 TLSv1.1 TLSv1.2;
    ssl_ciphers "HIGH:!aNULL:!MD5 or HIGH:!aNULL:!MD5:!3DES";
    ssl_prefer_server_ciphers on;
    try_files $uri $uri/ =404;
}

上述設定透過 uwsgi 在 127.0.0.1:8080 啟動 cmsimde/wsgi.py 動態網站程式, 並將網際資料送給 nginx 後, 以 port 443 ssl 協定對連外網路進行 WWW 伺服.

安裝 stunnel:

因為隨後安裝 Fossil SCM 時, 需要透過 stunnel 配置 https 協定連線, 在此先完成安裝, 並且與 uwsgi + nginx 共用 credential 與 key.

進入 /etc/stunnel 後, 執行下列指令, 建立 localhost.crt 與 localhost.key:

sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout localhost.key -out localhost.crt

設定 uwsgi_ini/uwsgi.ini:

[uwsgi]
socket = :8080
uid = wcm2021
gid = wcm2021
plugins-dir = /usr/lib/uwsgi/plugins/
plugin = python3
master = true
process = 4
threads = 2
chdir = /home/wcm2021/github/cmstemplate
wsgi-file = /home/wcm2021/github/cmstemplate/cmsimde/wsgi.py

測試 uwsgi 設定:

sudo /usr/bin/uwsgi --emperor /home/wcm2021/uwsgi_ini

在上述指令啟動時, CMSiMDE 出現無法 import flask app 的錯誤, 因此修改 CMSiMDE, 納入下列設定:

import sys
import os
sys.path.append(os.path.join(os.path.dirname(__file__)))
# import flaskapp at the same directory
import flaskapp
import ssl

即可順利透過 uwsgi 啟動 CMSiMDE.

apt proxy 設定檔案:

若採手動設定, apt 的 proxy 將位於 /etc/apt/apt.conf, 但先前曾在安裝階段輸入 proxy 設定後, Ubuntu 會將 apt 的 proxy 設定存入:

/etc/apt/apt.conf.d/90curtin-aptproxy

設定 Ubuntu 系統服務, 用來啟動 uwsgi:

/etc/systemd/system/ 目錄中的 cmsimde.service 服務啟動檔案內容:

[Unit]
Description=uWSGI to serve CMSiMDE
After=network.target
  
[Service]
User=wcm2021
Group=wcm2021
WorkingDirectory=/home/wcm2021/uwsgi_ini
ExecStart=/usr/bin/uwsgi --emperor /home/wcm2021/uwsgi_ini
  
[Install]
WantedBy=multi-user.target

接著將 cmsimde 服務設為隨系統開機啟動:

sudo systemctl enable cmsimde

若要取消 cmsimde 服務隨系統開機啟動:

sudo systemctl disable cmsimde

手動啟動 cmsimde.service 服務:

sudo systemctl start cmsimde

手動停止 cmsimde.service 服務:

sudo systemctl stop cmsimde

在 Ubuntu 虛擬主機中設定 Fossil SCM:

安裝 fossil:

sudo apt install fossil

查 fossil 版本:

fossil version

因為採用 sudo apt install fossil 的版本為 2.10 版, 希望採用目前的最新版本 2.15, 因此:

直接在 Windows 10 下載 https://fossil-scm.org/home/uv/fossil-linux-x64-snapshot-20210429.tar.gz, 解開壓縮檔案後, 將 fossil 送到 /homt/wcm2021/ 目錄下, 再使用指令, 將檔案複製到 /usr/bin/ 目錄下.

也可以直接在 Ubuntu 下以 wget 取下 https://fossil-scm.org/home/uv/fossil-linux-x64-snapshot-20210429.tar.gz 後, 使用 tar zxvf 解開壓縮檔後, 將檔案複製至 /usr/bin 中執行.

在 Ubuntu 要查詢 fossil 可執行檔案的位置, 使用 whereis fossil.

sudo cp fossil /usr/bin/

複製後, 再用 fossil version 查驗版本, 確認為最新的 2.15 版.

接著在 /home/wcm2021 目錄下建立 repository 目錄, 並且利用 fossil 起始一個 wcm2021.fossil 倉儲壓縮檔案.

mkdir repository

cd repository

fossil init cmstemplate.fossil

完成後, 紀錄 cmstemplate.fossil 的管理密碼, 以便啟動 Fossil SCM 伺服器時, 利用 wcm2021 當作帳號, 而以記錄的管理者密碼登入進行設定.

截至 Fossil SCM 2.15 版, 若 server 使用 IPv6 網路協定, 則必須設定 DNS domain 後, 全程使用 domain name 進行 fossil clone https://username@your_fossil_server_domain_name:port/cmstemplate cmstemplate.fossil, 表示要使用 fossil clone 將位於遠端伺服器中, 帳號為 username 且倉儲名稱為 cmstemplate 的檔案, clone 到近端存為 cmstemplate.fossil.

接下來為了要在 https 的模式下使用 Fossil SCM 伺服器, 因此必須安裝配置 stunnel:

sudo apt install stunnel

修改 /etc/environment, 加入:

HTTPS=on

修改 /etc/default/stunnel4, 修改 ENABLED=1

假如此時 /etc/stunnel 目錄下尚無 localhost.crt 與 localhost.key, 可以利用下列指令建立:

建立 localhost.key 與 localhost.crt:

sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout localhost.key -out localhost.crt

接著設定 /etc/stunnel/stunnel.conf 如下:

[https]
accept = :5443
accept = :::5443
cert = /etc/stunnel/localhost.crt
key = /etc/stunnel/localhost.key
exec = /usr/bin/fossil
execargs = /usr/bin/fossil http /home/wcm2021/repository/ --https --nojail --notfound wcm2021

表示利用 stunnel 執行 fossil 指令, 並且透過 http 協定啟動 位於 /home/wcm2021/repository 目錄下的倉儲壓縮檔案, 且附加採用 https 與 nojail 模式啟動.

其中 --nojail 目的在 drop the root privilege but do not enter the chroot jail, 其後的 --notfound 表示若沒有特別在 URL 中列出所要擷取的 .fossil 檔案 (指位於 /home/wcm2021/repository/ 目錄下), 則採用 wcm2021.fossil

重新啟用 stunnel:

sudo /etc/init.d/stunnel4 restart

啟動後的 Fossil SCM 位於 port 5443, 且將 wcm2021 管理者密碼設為 XXXX.

Fossil SCM 查詢 .fossil 檔案內容:

以 fossil sql 指令擷取 .fossil 檔案內容

使用者以 fossil init foo.fossil 建立倉儲檔案後, 除了使用 sqlite3 命令檢視 fossil 倉儲內容外, 也可以利用 fossil 指令查驗倉儲資料庫檔案的內容.

fossil sql -R foo.fossil

進入 sqlite 指令環境後, 可以利用 .schema user 查驗 user 資料表的欄位設計.

以 select login, pw, cap from user;

可以查詢目前 foo.fossil 檔案中 user 資料表 login, pw 與 cap 等3個欄位的資料內容.

若要更改特定 login 登入帳號的 pw 或 cap 可以使用下列 sqlite3 SQL 語法:

update user set pw='mynewpassword' where login='myloginaccount';

update user set cap='s' where login='myloginaccount';

使用 https://doc.qt.io/qt-5/dnd.html drag and drop 方式, 將 STL 零件檔案帶進特定座標下的 viewer 視窗.

from PyQt5.QtWidgets import QMainWindow, QApplication
import sys


class MainWidget(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Drag and Drop")
        self.resize(720, 480)
        self.setAcceptDrops(True)

    def dragEnterEvent(self, event):
        if event.mimeData().hasUrls():
            event.accept()
        else:
            event.ignore()

    def dropEvent(self, event):
        files = [u.toLocalFile() for u in event.mimeData().urls()]
        for f in files:
            print(f)


if __name__ == '__main__':
    app = QApplication(sys.argv)
    ui = MainWidget()
    ui.show()
    sys.exit(app.exec_())
