Fossil SCM 研究

Git 的出現是為了有效管理 Linux 核心程式碼, 而 Fossil SCM 則是為了管理 SQLite 原始碼而生, Fossil SCM 的作者稱選擇 Git 或 Fossil SCM 來管理你所開發的程式碼, 端看你所開發的程式碼比較類似 Linux 核心程式或者比較接近 SQLite.

Linux 是允許多人維護的核心, 強調任何人都能夠貢獻特定功能的原始碼, 至於 SQLite 則採行嚴格的封閉式開發模式, 只允許幾個人參與開發.

搜尋 "Fossil SCM" 只傳回 23000 筆, 對於已經存在超過 15 年的套件而言, 確實屬於冷門項目, 若用曲高和寡來形容不知道恰不恰當.

以下是與 Fossil SCM 有關的參考資料:

鏡射 Fossil SCM 倉儲至 Github: https://www.fossil-scm.org/home/doc/trunk/www/mirrortogithub.md

Fossil SCM serving by CGI: https://www.fossil-scm.org/home/doc/trunk/www/server/any/cgi.md

Fossil SCM 簡介: http://mde.tw/2017springcd/blog/intro-fossil-scm.html

搭建 Fossil SCM server:

http://mde.tw/wcm2021/blog/fossil-scm-on-ubuntu.html

Fossil Clone:

fossil clone https://user_name@fossil_server_url/fossil_repo_name fossil_repo_name.fossil

表示從 fossil_server_url 伺服器, 使用 user_name 為帳號, clone fossil_repo_name 倉儲, 在近端存為 fossil_repo_name.fossil

在近端解開 fossil_repo_name.fossil:

mkdir wd

cd wd

mkdir fossil_repo_name

cd fossil_repo_name

fossil open ./../../fossil_repo_name.fossil

在此目錄中對倉儲改版

fossil add .

fossil commit -m "your commit message"

fossil 會自動將改版內容推向 ./../../ 中的 .fossil 檔案, 以及 fossil_server_url 上的倉儲.

假如您有一台實體或虛擬的主機安裝著 Ubuntu 操作系統, 手上只剩下 IPv6 網址能夠配置, 以下將說明如何建立一台合用的 Fossil SCM 伺服器, 可以用來管理 CMSiMDE 靜態與動態網站, 同時, 若採用 Desktop GUI 介面, 之於機械設計相關應用, 可用來配置諸如 RoboDK, CoppeliaSim 以及 Webots 等機電模擬與控制系統設計相關套件. 最終目的是希望透過網際虛實整合模式, 在多人協同設計的架構下開發機電資產品.

第一步: 安裝一台 Ubuntu Desktop 虛擬主機.

首先下載 Ubuntu Desktop iso 與 Virtualbox Windows 套件.

假如你是在系上的網路主幹, 可以從 http://a.kmol.info:88/ubuntu-20.04.2.0-desktop-amd64.iso 下載, 否則, 可以從 https://ubuntu.com/download/desktop 下載, desktop 的 iso 檔案將近 3 GB 左右.

而 Virtualbox 則可直接從 https://www.virtualbox.org/wiki/Downloads 下載.

接下來要在 Windows 10 中完成 Virtualbox 的安裝, 然後啟動, 準備利用 ubuntu-20.04.2.0-desktop-amd64.iso 來安裝一台 Virtualbox 平台上的虛擬主機.

配置虛擬主機之前, 必須要知道目前架設 Fossil SCM 的目的是讓其他組員能夠透過網路將特定專案的資料版本送至 Fossil SCM server 進行管理, 因此虛擬主機必須透過 Bridged Network 網路的配置或者採 Host Only 以及 Port Mapping 的方式, 讓外部網路封包可以進入虛擬主機.
