期中專案

之前的 git 倉儲都是先在 Github 網際平台上建立後, 再設法 git clone 到近端改版, 但是這樣的前提是在 Github 建立倉儲時, 至少新增一個檔案 README.md,  但是假如在 Github 建立一個空倉儲則通常會建議使用者先在近端以 git init 產生一個空的 git 倉儲後, 納入改版資料然後再透過 git remote add 設定與遠端的 Github 倉儲 URL 進行對應.

而通常這樣的 git remote add 會以 origin 作為代號.

現在假如連倉儲的 origin 都要自行設定, 其步驟如下:

  1. 在近端命令視窗中, 以 git init 建立空的倉儲, 取得 Github 空倉儲的 URL 後, 可以採用 https 或 SSH 模式進行 git remote add 的設定.
  2. 假如遠端的 Github 倉儲使用者帳號為 mdecourse, 而空倉儲名稱為 wcm_midterm, 則其 https 的 URL 為 https://github.com/mdecourse/wcm_midterm, 若採用 SSH 則其 URL 為 git@my_putty_session:mdecourse/wcm_midterm
  3. 步驟 2 的指令為 git remote add https://github.com/mdecourse/wcm_midterm.git origin, 或者使用 git remote add git@my_putty_session:mdecourse/wcm_midterm.git origin
  4. 另外若將此一倉儲在近端使用 Heroku 的指令與遠端的 Heroku 名稱為 wcm_midterm 的 app 進行對應設定, 則命令在 heroku login -i 之後, 以 heroku git:remote -a wcm_midterm, 之後就可以透過 git push heroku 將此一倉儲的版本推向 Heroku.
